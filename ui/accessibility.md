# 접근성 (Accessibility)

모든 UI는 **마우스 없이도, 시각 없이도, 느린 모션을 선호해도** 동작해야 한다. 접근성은 사후 보정이 아니라 **설계 단계부터** 반영한다.

## 시맨틱 HTML 우선

- 역할이 있는 요소는 시맨틱 태그를 사용한다. div로 전부 그리지 않는다.
  - 버튼은 `<button>`, 링크는 `<a>`, 폼 입력은 `<input>`/`<label>`
  - 페이지 구조는 `<header>`, `<nav>`, `<main>`, `<aside>`, `<footer>`
- ARIA는 **시맨틱 HTML로 표현 불가능할 때만** 사용한다. (First rule of ARIA)

## ARIA 기본 규칙

- `role`은 필요할 때만 부여. 이미 의미 있는 요소에 `role`을 덮어쓰지 않는다.
- 동적 상태는 ARIA 속성으로 반드시 표현한다.
  - 열림/닫힘: `aria-expanded`
  - 선택 상태: `aria-selected`, `aria-checked`
  - 비활성: `aria-disabled` (DOM `disabled`를 못 쓸 때)
  - 현재 위치: `aria-current="page"` 등
  - 라이브 영역: `aria-live="polite"` / `"assertive"`
- 레이블 없는 아이콘 버튼은 **반드시** `aria-label` 제공.

## 키보드 내비게이션

**모든 인터랙션은 키보드만으로 가능해야 한다.**

| 요소 | 필수 키 동작 |
|---|---|
| 버튼/링크 | `Enter`, `Space` |
| 체크박스/스위치 | `Space` |
| 라디오 그룹 | `↑` `↓` `←` `→` |
| 탭 | `←` `→`, `Home` `End` |
| 메뉴/셀렉트 | `↑` `↓`, `Esc` 닫기, `Enter` 선택 |
| 다이얼로그 | `Esc` 닫기, 포커스 트랩 |

- 포커스 순서는 **시각적 순서와 일치**해야 한다.
- 키보드 포커스 링을 **절대 없애지 않는다**. 디자인 요구 시 `:focus-visible`로 조건부 표시.

## 포커스 관리

- **모달·다이얼로그 열릴 때**: 첫 포커스 가능 요소로 포커스 이동, 포커스 트랩 적용.
- **모달 닫힐 때**: 열기 트리거 요소로 포커스 복귀.
- 동적으로 제거되는 요소에 포커스가 있었다면 **의미 있는 대체 위치**로 이동.
- `autoFocus`는 신중히. 사용자 의도와 어긋나면 혼란.

## 스크린리더 고려

- **이미지**: 의미 있는 이미지엔 `alt` 필수, 장식용은 `alt=""`.
- **아이콘**: 텍스트와 함께 쓰이면 `aria-hidden="true"`. 단독 의미 전달 시 `aria-label`.
- **라이브 업데이트**: 토스트·알림은 `aria-live` 또는 `role="status"` / `"alert"`.
- **폼 에러**: 입력 필드와 에러 메시지를 `aria-describedby`로 연결.

## 색과 대비

- 텍스트 대비: **WCAG AA (4.5:1)** 최소 준수. 큰 텍스트는 3:1.
- **색상 하나로만 의미 전달 금지**. (예: 빨간색 = 에러만으로는 부족, 아이콘·텍스트 병행)
- 포커스/호버/활성 상태는 **색 + 다른 시각 단서**(테두리, 굵기, 아이콘)로 중복 표시.

## 모션

- **`prefers-reduced-motion`** 쿼리를 존중한다.
- 자동 재생되는 애니메이션·전환은 reduced-motion 환경에서 비활성 또는 단축.
- 1초 이상 반복되는 애니메이션은 **정지 수단**을 제공한다. (WCAG 2.2.2)

```
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 터치 대상

- 터치 가능한 요소는 **최소 44×44px** (WCAG 2.5.5). 모바일 특히 엄수.
- 인접한 터치 타겟은 충분한 간격 확보.

## 폼

- 모든 입력은 `<label>`과 연결(`for`/`id` 또는 감싸기).
- placeholder를 label 대용으로 쓰지 않는다.
- 필수 필드는 `required` + 시각 표시 둘 다.
- 에러 메시지는 **어떤 필드가 왜 틀렸는지** 명확히. "입력값이 잘못되었습니다" 금지.

## 테스트 원칙

- 새 컴포넌트를 만들면 최소한 **키보드만으로 전 기능 조작** 가능한지 확인.
- 스크린리더(VoiceOver/NVDA)로 최소 한 번 읽혀본다.
- axe, Lighthouse 등 자동화 도구로 정기 감사.

## 금지

- **`tabindex` 양수 사용** (`tabindex="1"` 등) — DOM 순서와 불일치 유발.
- **`outline: none`** 단독 적용 — 대체 포커스 스타일 없으면 금지.
- **키보드 이벤트만 처리**하고 마우스만 처리하거나, **그 반대**. 둘 다 지원.
- **마우스 호버에만 있는 정보** — 터치·키보드 사용자 접근 불가.
